Registration Email Domain Check module

Created by: DB and Yankidank
Last Updated: 02/03/2011
Created on: 01/24/2011

Description

This module will allow admins to set up a whitelist or blacklist for allowed email registration domains.

Settings and Examples

This module has two different methods: whitelist and blacklist modes. The blacklist mode will reject any registrations from email addresses, domains, or partial word matches that you set up. The whitelist does the opposite and will only accept registrations from a list of allowed phrases. The module only checks the ending part of an email address, so if you enter the word "test" as a blacklist word it would block foo@bar.sampletest, but not foo@test.bar.

Blacklist Examples

  • Adding @mailinator.com would block only email addresses ending with "@mailinator.com".
  • Adding ".ru" would ban foo@bar.co.ru, but not foo@bar.ru.co
  • Adding "bar" would only block domains ending with "bar" (ex. the address email@foo.bar)

Whitelist Examples

  • Adding "@company.com" would only allow addresses ending with @company.com when using the whitelist method.
  • Adding ".edu" would only allow .edu addresses (plus other exceptions added to the list)
